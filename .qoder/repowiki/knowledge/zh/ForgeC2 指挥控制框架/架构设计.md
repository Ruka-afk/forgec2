- **分层架构**：采用 `cmd/server` 作为入口，`internal/server` 处理 HTTP/TCP 请求与业务逻辑，`internal/db` 负责数据持久化，`internal/payload` 实现 Agent 源码生成与编译。
- **通信设计**：支持 HTTPS (Gin) 和原生 TCP (长度前缀 JSON) 双传输层；Beacon 逻辑在 `processBeacon` 中统一处理，适配 HTTP 与 TCP 上下文。
- **安全边界**：通过 `internal/server/middleware` 实现 JWT 会话认证、CSRF 防护（双重提交 Cookie）、速率限制及安全响应头注入。
- **数据流**：Web UI 通过 HTMX/WebSocket 与后端交互；Agent 通过 `/api/v1/beacon` 或 TCP 端口上报状态并拉取任务；任务结果经 Base64 解码后存入 SQLite 或文件系统。
